How this works

The scanner compares each prop’s fair price—the price that would reflect win odds with the sportsbook margin removed—to the price available to bet. When fair probability supports a better payout than the line you can take, the opportunity is flagged as positive expected value (+EV).

Market

Loads every supported category in one request from Positive EV (BetMarket=0). This dropdown only filters the table in your browser—changing it does not refetch. Game/team moneyline, game total, 1st-inning runs, team totals, batter doubles/triples, and pitcher-to-record-win rows are excluded.

Devig method

Each side of a two-way market carries an implied win probability; together they exceed 100% because of the book’s hold. Removing that hold—devigging—produces fair win chances for each side. The method you select determines how that adjustment is performed. Results are expressed as fair American odds and drive Expected Value.

  • Multiplicative — Scales both implied probabilities down proportionally so they sum to 100%. The relative shape of the market is preserved; this approach is standard and straightforward to interpret.
  • Additive — Estimates total overround, subtracts half from each side’s implied probability, then renormalizes. Delivers a symmetric response to the book’s margin.
  • Probit — Maps implied probabilities through the normal distribution, applies a symmetric adjustment, and maps back.
  • Shin — Applies a Shin-style (Zangwill) correction so adjusted probabilities respect the posted market shape while summing to 100%.
  • Power — Raises each implied probability to a calibrated power until the pair sums to 100%.
  • Worst case — Evaluates multiple reasonable fair probabilities and uses the one that gives your side the least edge.
  • Average — Computes fair probabilities under several methods, then averages the fair probability for the side you are analyzing.

Best book

When set to a specific sportsbook, the main table only lists props where that book posts the best price for the line. All rows shows every prop that passes the other filters, regardless of which book is best.

Restricts which sportsbooks feed the two-way prices used in devigging. When set to all books, every available line in scope is eligible. Use Devig books… to pick multiple books and optional weights for consensus fair.

Ballpark Pal park factors (table columns)

The HR, 2B/3B, 1B, and Runs columns show combined effect park factors from Ballpark Pal, matched to each game on your board. They are informational context for the slate, not inputs to the EV math in this app.

Model and data: Ballpark Pal describes these as coming from a machine-learning model trained on a very large set of batted balls and games, with the goal of isolating how stadium and weather interact at each venue.

How to read the numbers: The percentages are expressed relative to the league-wide MLB average for the season—not relative to that park’s own long-run baseline.

Caveat: Park effects are small and noisy in any one game. Treat these columns as context, not a guarantee.

Bankroll ($)

Converts the Kelly stake from a fraction of bankroll into dollars in the Kelly column. Filter logic and line selection are unchanged.

Boost

When a profit boost (or similar promo) increases winnings on a winning ticket, pick the level so EV %, Kelly, and Implied use the boosted payout. Fair value stays the pre-promotion benchmark from the market. No Sweat is approximated for modeling.

Main table

  • Expected Value: Modeled edge versus fair, in percent, at the effective best price (including any boost).
  • Kelly: Full-Kelly dollar suggestion for the stated bankroll—illustrative sizing only.
  • Best book: Operator posting the top price for that line; odds can be hidden with the header toggle.
  • Player, Line, Prop, O/U: Participant, threshold, market label, and side.
  • Fair: Fair American odds after devig; excludes promotions.
  • CS*: Ballpark Pal consensus (American odds) from the source page.
  • Implied: Win probability implied by the price used for that row, after boost if applicable.
  • Δ (vig): Probability delta between the outlier book and the market consensus (with vig), when present on the Ballpark Pal table.
  • Matchup / park columns: Teams and Ballpark Pal combined-effect park factors.
  • Additional columns list each book’s price for side-by-side comparison.

Odds come from Positive EV (all markets per day). Rows need at least two books on the same line (server default). Optional: set MLB_SCANNER_ODDS_SCREEN=1 to also merge Odds Screen (much slower).
